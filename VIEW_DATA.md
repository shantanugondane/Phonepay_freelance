# How to View Your MongoDB Data

## Where Data is Stored

### Local MongoDB (Your Current Setup)
- **Database Name:** `phonepe_portal`
- **Data Directory:** `C:\Program Files\MongoDB\Server\8.2\data\`
- **Collection:** `users` (contains all registered users)

---

## How to View Your Data

### Option 1: Using MongoDB Compass (Easiest - GUI)

1. **Open MongoDB Compass** (you installed it earlier)

2. **Connect to Local MongoDB:**
   - Click "Add new connection" or "New Connection"
   - Connection String: `mongodb://localhost:27017`
   - Or just click "Connect" (it should auto-detect)

3. **Navigate to Your Database:**
   - In the left sidebar, you'll see databases
   - Click on `phonepe_portal` database
   - Click on `users` collection
   - You'll see all your registered users!

4. **View User Data:**
   - You'll see documents (rows) for each user
   - Each document contains: email, name, role, _id, timestamps, etc.
   - Password is hashed (encrypted) - you won't see the actual password

---

### Option 2: Using MongoDB Shell (Command Line)

1. **Open Command Prompt/Terminal**

2. **Connect to MongoDB:**
   ```bash
   mongosh
   ```

3. **Switch to your database:**
   ```javascript
   use phonepe_portal
   ```

4. **View all users:**
   ```javascript
   db.users.find().pretty()
   ```

5. **Count users:**
   ```javascript
   db.users.countDocuments()
   ```

6. **Find specific user:**
   ```javascript
   db.users.findOne({ email: "admin@phonepe.com" })
   ```

---

### Option 3: View via API (Programmatic)

You can also query via your backend API (we'll add this later).

## What Data is Stored

Each user document looks like this:
```json
{
  "_id": "6964067446d7acd6b226d658",
  "email": "admin@phonepe.com",
  "name": "Admin User",
  "role": "admin",
  "password": "$2a$10$...",  // Hashed (encrypted) password
  "isActive": true,
  "createdAt": "2024-01-15T10:30:00.000Z",
  "updatedAt": "2024-01-15T10:30:00.000Z"
}
```

**Note:** 
- Password is hashed using bcrypt (you can't see the actual password)
- `_id` is auto-generated by MongoDB
- Timestamps are automatically managed
